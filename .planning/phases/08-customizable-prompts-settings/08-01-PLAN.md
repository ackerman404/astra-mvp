# Plan 08-01: YAML Config and GUI Controls

## Context

Phase 8 makes all LLM prompts, job context, and tone settings configurable via YAML.

**Goal**: Users can customize prompts without editing code, set job context for relevant answers, and select tone style.

**Files to modify**: requirements.txt, config.py, rag.py, gui.py

## Tasks

### Task 1: Add PyYAML dependency
**File**: requirements.txt
**Action**: Add `pyyaml>=6.0.0` to dependencies

### Task 2: Extend config.py with YAML functions
**File**: config.py
**Actions**:
- Add `import yaml`
- Add `get_prompts_config_path()` - returns `~/.config/astra/prompts.yaml`
- Add `get_default_prompts_config()` - returns default config dict with all prompts
- Add `load_prompts_config()` - loads YAML, creates default if missing, handles errors
- Add `save_prompts_config(config)` - saves config to YAML file

### Task 3: Refactor rag.py to use config
**File**: rag.py
**Actions**:
- Rename existing prompts to `DEFAULT_*` constants (keep as fallbacks)
- Add `_prompts_config = None` cache variable
- Add `reload_prompts_config()` - force reload from YAML
- Add `get_prompt(name)` - get prompt by name with fallback
- Add `get_tone_instruction(tone)` - get tone text from config
- Add `get_default_job_context()` - return configured default
- Add `get_default_tone()` - return configured default tone
- Add `get_available_tones()` - list all tone names from config
- Update `classify_utterance()` to use `get_prompt("classification")`
- Update `ask_bullets()` to use `get_prompt("bullet_system")`
- Update `ask_script()` to use `get_prompt("script_system")` and `get_tone_instruction(tone)`

### Task 4: Add GUI controls to main window
**File**: gui.py
**Actions**:
- Import `get_available_tones`, `get_default_job_context`, `get_default_tone`, `reload_prompts_config` from rag
- In `_init_ui()`, after auto-answer toggle section, add settings frame:
  - QLabel "Job Context:" + QLineEdit (placeholder: "e.g., Senior SAP MM Consultant")
  - QLabel "Tone:" + QComboBox (populated from `get_available_tones()`)
  - QPushButton "Reload Config" connected to `_on_reload_config()`
- Add `_on_reload_config()` method:
  - Call `reload_prompts_config()`
  - Refresh tone dropdown options
  - Show status message "Config reloaded"
- Update `_generate_parallel()` to pass `job_context` and `tone` from UI to generation functions

### Task 5: Update generation functions to accept job_context and tone
**File**: rag.py
**Actions**:
- Update `ask_bullets(question, job_context=None)` to inject job_context into prompt
- Update `ask_script(question, tone="professional", job_context=None)` to accept job_context
- Both functions format job_context into system prompt if provided

## Verification

- [ ] `pyyaml` in requirements.txt
- [ ] First run creates `~/.config/astra/prompts.yaml` with defaults
- [ ] App loads and runs with default config
- [ ] Job context input visible in main window
- [ ] Tone dropdown shows professional, casual, confident
- [ ] Reload Config button refreshes settings
- [ ] Invalid YAML gracefully falls back to defaults
- [ ] Generation uses job_context and tone from UI

## Rollback

If issues occur:
1. Revert rag.py to use hardcoded prompts directly
2. Remove GUI controls from gui.py
3. Config functions can remain (no harm if unused)
