<task type="auto">
  <name>Task 1: Implement chat completions proxy endpoint with SSE streaming</name>
  <files>backend/proxy.py, backend/main.py</files>
  <action>
    **backend/proxy.py** — Core proxy endpoint:

    Create `POST /v1/chat/completions` endpoint that mirrors OpenAI's API shape. This allows the desktop app to use `OpenAI(base_url=proxy_url, api_key=license_key)` with zero custom HTTP code.

    **Request flow:**
    1. License key validated via `Depends(validate_license)` from auth.py (already built in 09-01)
    2. Parse request body as JSON (use `Request.json()`, not a Pydantic model — pass through to OpenAI as-is to avoid schema maintenance)
    3. **Model whitelist enforcement (PROXY-09):** Check `body["model"]` is in `settings.ALLOWED_MODELS`. If not, return 400 with `{"error": {"code": "model_not_allowed", "message": "Model not permitted. Allowed: gpt-4o, gpt-4o-mini"}}`.
    4. Get `AsyncOpenAI` client from `request.app.state.openai_client`
    5. Determine if streaming: `is_streaming = body.get("stream", False)`

    **Streaming path (PROXY-01):**
    - Call `openai_client.chat.completions.create(**body)` — pass the body dict directly. The AsyncOpenAI SDK handles the streaming.
    - Important: Add `stream_options={"include_usage": True}` to the body if streaming, so the final chunk includes token usage (needed for PROXY-08 logging).
    - Create an async generator that yields SSE-formatted chunks: `f"data: {chunk.model_dump_json()}\n\n"` for each chunk, then `"data: [DONE]\n\n"` at the end.
    - Track token usage from the final chunk's `usage` field.
    - Return `StreamingResponse(generator, media_type="text/event-stream", headers={"Cache-Control": "no-cache", "X-Accel-Buffering": "no"})`.
    - Wrap generator in try/finally to handle cleanup if client disconnects mid-stream.

    **Non-streaming path:**
    - Call `openai_client.chat.completions.create(**body)` without stream=True
    - Extract token usage from `response.usage`
    - Return `response.model_dump()` as JSON

    **Retry logic (PROXY-07):**
    - Wrap the OpenAI call in retry logic: if OpenAI returns 429 (rate limit) or 500 (server error), wait 2 seconds and retry once.
    - Use try/except for `openai.RateLimitError` and `openai.InternalServerError`.
    - If retry also fails, return the error to the client.

    **Timeout handling (PROXY-05):**
    - The `AsyncOpenAI` client should be created with `timeout=httpx.Timeout(connect=5.0, read=60.0, write=10.0, pool=5.0)` in the lifespan (update main.py).
    - On timeout, return 504 with `{"error": {"code": "timeout", "message": "Request timed out. Please try again."}}`.

    **Error mapping (PROXY-03):**
    - Catch OpenAI SDK exceptions and map to user-friendly responses:
      - `openai.AuthenticationError` → 502 `{"error": {"code": "upstream_auth", "message": "AI service configuration error. Contact support."}}`  (never expose that it's an API key issue)
      - `openai.RateLimitError` → 429 `{"error": {"code": "rate_limited", "message": "Service is busy. Please wait a moment."}}`
      - `openai.APIError` → 502 `{"error": {"code": "upstream_error", "message": "AI service temporarily unavailable."}}`
      - `openai.APIConnectionError` → 502 `{"error": {"code": "upstream_unreachable", "message": "Cannot reach AI service. Try again shortly."}}`
      - `openai.APITimeoutError` → 504 `{"error": {"code": "timeout", "message": "Request timed out. Please try again."}}`

    **Usage logging (PROXY-08):**
    - After each successful request (streaming or non-streaming), insert a `UsageLog` record with: license_key_id, endpoint ("chat/completions"), model, prompt_tokens, completion_tokens, status_code (200), latency_ms (measure with `time.monotonic()`).
    - For streaming, token counts come from the final chunk with `stream_options.include_usage`.
    - Do NOT block the response to write the log — fire-and-forget using `asyncio.create_task()` or write after response completes.

    Register the proxy router in `backend/main.py` via `app.include_router(proxy_router)`.

    **Critical pitfall avoidance (from PITFALLS.md P-03):**
    - Use async generators only (never sync generators with StreamingResponse)
    - Reuse the AsyncOpenAI client from app.state (do NOT create per-request)
    - Set `X-Accel-Buffering: no` header on streaming responses
    - Handle client disconnect gracefully in the generator
  </action>
  <verify>
    Start server: `cd backend && uvicorn main:app --port 8000`
    Generate a test key and activate it via CLI.
    Test with curl:
    ```

    ```
    Should see SSE chunks streaming back. Test with invalid key — should get 403.
    Test with disallowed model — should get 400.
  </verify>
  <done>
    Proxy forwards both streaming and non-streaming requests to OpenAI. License key validated on every request. Model whitelist enforced. Errors mapped to structured JSON. Usage logged per request. Retry on OpenAI 429/500.
  </done>

<task type="auto">
  <name>Task 2: Add health check, rate limiting, request logging, and startup validation</name>
  <files>backend/middleware.py, backend/main.py</files>
  <action>
    **Health check endpoint (REL-01):**
    - `GET /health` — Returns `{"status": "ok", "version": "3.0.0"}`. No auth required.
    - Also check OpenAI reachability by calling `openai_client.models.list()` with a short timeout (5s). If reachable, add `"openai": "reachable"`. If not, add `"openai": "unreachable"`. Do NOT fail the health check if OpenAI is down — the proxy itself is still healthy.

    **Startup validation (REL-05):**
    - In the FastAPI lifespan startup, after creating the AsyncOpenAI client, make a test call to `openai_client.models.list()`.
    - If it fails with AuthenticationError, print a clear error message: `"FATAL: OPENAI_API_KEY is invalid. Server refusing to start."` and call `sys.exit(1)`.
    - If it fails with a connection error, print a warning but allow startup (OpenAI might be temporarily down).

    **backend/middleware.py** — Rate limiting and logging:

    **Rate limiting (PROXY-06):**
    - Implement in-memory per-key rate limiting using a dict: `{key_id: [(timestamp, ...)]}`
    - Create a FastAPI dependency `check_rate_limit(license_key: LicenseKey)` that:
      - Counts requests in the last 60 seconds for this key
      - If count >= `RATE_LIMIT_COMPLETIONS_RPM` (20), raise HTTPException 429 with `{"error": {"code": "rate_limited", "message": "Too many requests. Please wait.", "retry_after": N}}` where N is seconds until oldest request expires from the window
    - Clean up old entries (> 60s) on each check to prevent memory leak
    - Wire this dependency into the proxy endpoint via `Depends(check_rate_limit)` — it should run AFTER license validation

    **Request logging (REL-04):**
    - Add FastAPI middleware that logs each request:
      - Timestamp (ISO format)
      - License key (first 8 chars only — truncated for privacy)
      - Endpoint path
      - HTTP method
      - Status code
      - Latency (ms)
    - Use Python `logging` module with structured format (not print statements)
    - Log level: INFO for successful requests, WARNING for 4xx, ERROR for 5xx

    **Structured error handling (REL-03):**
    - Add a global exception handler via `@app.exception_handler(Exception)` that catches any unhandled exception and returns `{"error": {"code": "internal_error", "message": "Something went wrong. Please try again."}}` with status 500.
    - NEVER return stack traces to the client (log them server-side only).
    - Add specific handler for `RequestValidationError` to return 400 with `{"error": {"code": "invalid_request", "message": "..."}}`.

    Wire all middleware and dependencies into `backend/main.py`.
  </action>
  <verify>
    Start server and test:
    - `GET /health` returns 200 with status and OpenAI reachability
    - Send 25 rapid requests with same key — first 20 succeed, remaining get 429
    - Check server logs show structured request entries with truncated key
    - Send request with malformed JSON — get 400 structured error (not stack trace)
    - Remove OPENAI_API_KEY from env and restart — server should refuse to start
  </verify>
  <done>
    Health check reports server and OpenAI status. Rate limiting enforced at 20 RPM per key. All requests logged with key, endpoint, status, latency. All errors return structured JSON. Server validates OpenAI key on startup.
  </done>
</task>
