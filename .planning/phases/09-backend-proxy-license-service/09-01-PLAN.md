<task type="auto">
  <name>Task 2: Implement license key validation endpoints and management CLI</name>
  <files>backend/auth.py, backend/license_cli.py, backend/main.py (add routes)</files>
  <action>
    **backend/auth.py** — License validation:
    - `validate_license` async dependency function:
      - Extract key from `Authorization: Bearer <key>` header using `fastapi.security.HTTPBearer`
      - Look up key in database by exact match (NOT hashed — UUID v4 keys are long enough, hashing adds complexity for basic deterrent)
      - Check status is "active" (reject "unused", "revoked" with specific messages)
      - Check `expires_at` is None or in future
      - Update `last_validated_at` to now
      - Return the `LicenseKey` object for downstream use
      - Raise `HTTPException(401)` for missing/malformed auth, `HTTPException(403)` for invalid/revoked/expired keys with structured JSON body: `{"error": {"code": "invalid_key"|"revoked_key"|"expired_key", "message": "..."}}`
    - Use `hmac.compare_digest()` for key comparison to prevent timing attacks

    - License HTTP endpoints (add to main.py or separate router):
      - `POST /v1/license/activate` — Accepts JSON `{"license_key": "...", "hardware_id": "..."}`. Looks up key, checks status is "unused", sets status to "active", stores hardware_id, sets activated_at. Returns `{"valid": true, "tier": "...", "grace_period_days": 7}`. If key already active with different hardware_id, return 403 "Key is active on another machine."
      - `POST /v1/license/deactivate` — Accepts JSON `{"license_key": "...", "hardware_id": "..."}`. Verifies hardware_id matches, sets status back to "unused", clears hardware_id. Returns `{"deactivated": true}`.
      - `POST /v1/license/validate` — Accepts license key via Authorization header. Validates key and returns `{"valid": true, "tier": "...", "expires_at": ..., "grace_period_days": 7}`. Lightweight check without making LLM call. Used by desktop app for periodic revalidation.

    **backend/license_cli.py** — Runnable as `python -m backend.license_cli` or `python backend/license_cli.py`:
    - Use `argparse` with subcommands:
      - `generate --count N [--tier standard|premium] [--email EMAIL]` — Generate N UUID v4 keys, insert into DB with status "unused". Print generated keys to stdout.
      - `list [--status unused|active|revoked]` — List all keys with columns: key (truncated to first 8 chars + ...), status, tier, email, activated_at, hardware_id (truncated). Format as a clean table.
      - `activate KEY` — Set key status to "active" (admin override, no hardware_id required)
      - `deactivate KEY` — Set key status to "unused", clear hardware_id
      - `revoke KEY` — Set key status to "revoked" (permanent, cannot be reactivated)
    - Each command creates its own database session, commits, and prints confirmation
    - Import database and models from sibling modules

    Wire the license endpoints into `backend/main.py` by including the router.
  </action>

  </verify>
