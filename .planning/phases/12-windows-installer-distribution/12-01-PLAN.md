<task type="auto">
  <name>Task 1: Update PyInstaller spec for --onedir and fix hidden imports</name>
  <files>astra.spec, build_windows.bat</files>
  <action>
  Update astra.spec to use --onedir bundling instead of --onefile:

  1. **Switch to --onedir (COLLECT mode):**
     - Replace the single EXE block that bundles binaries/zipfiles/datas with a separate EXE + COLLECT structure
     - EXE should only contain `pyz` and `a.scripts` (the launcher)
     - COLLECT gathers `a.binaries`, `a.zipfiles`, `a.datas` into `dist/Astra/` directory
     - This avoids AV false positives common with --onefile and reduces startup time (no temp extraction)

  2. **Add missing hidden imports for v3.0 dependencies:**
     - `platformdirs` — config directory resolution
     - `requests` — license activation HTTP calls
     - `yaml` — prompts config (import name for pyyaml)
     - `rank_bm25` — hybrid search
     - `audio_capture` — local module (add to hiddenimports since it's imported dynamically)
     - `dotenv` — python-dotenv
     - `hashlib`, `hmac` — may need explicit inclusion for some builds
     - Keep existing: PyQt6.*, chromadb, openai, numpy, pdfplumber, pyaudiowpatch

  3. **Add collect_data_files for chromadb:**
     - chromadb has ONNX models and migration files that need bundling
     - Add: `datas += collect_data_files('chromadb')`

  4. **Add version info and icon placeholder:**
     - Set `version='1.0.0'` in EXE if supported, or add version_info tuple
     - Keep icon=None with comment for future icon file

  5. **Set console=False** (already set, verify it stays)

  6. **Update build_windows.bat:**
     - After PyInstaller completes, check for Inno Setup compiler (iscc.exe)
     - If iscc.exe found, compile installer/astra_setup.iss
     - Output installer to dist/AstraSetup.exe
     - Add error handling for each step
     - Print final output location

  Do NOT change the entry point (main.py) or add any new Python files.
  Do NOT add UPX compression flags — UPX triggers more AV false positives than it saves in size.
  </action>
  <verify>
  - astra.spec contains COLLECT() call (--onedir mode)
  - astra.spec hiddenimports list includes: platformdirs, requests, yaml, rank_bm25, audio_capture, dotenv
  - build_windows.bat references iscc for Inno Setup compilation
  - No syntax errors in .spec file (valid Python)
  </verify>
  <done>
  PyInstaller spec configured for --onedir bundling with all v3.0 hidden imports.
  Build script chains PyInstaller and Inno Setup compilation.
  </done>
</task>

<task type="auto">
  <name>Task 2: Create Inno Setup installer script</name>
  <files>installer/astra_setup.iss</files>
  <action>
  Create `installer/astra_setup.iss` — the Inno Setup script that packages PyInstaller's --onedir output into a standard Windows installer.

  1. **[Setup] section:**
     - AppName=Astra Interview Copilot
     - AppVersion=3.0
     - AppPublisher=Astra
     - DefaultDirName={localappdata}\Astra (per-user install, no admin)
     - DefaultGroupName=Astra
     - PrivilegesRequired=lowest (INST-06: no admin rights)
     - OutputDir=..\dist
     - OutputBaseFilename=AstraSetup
     - Compression=lzma2/ultra64
     - SolidCompression=yes
     - SetupIconFile= (comment placeholder for future .ico)
     - UninstallDisplayIcon={app}\Astra.exe
     - WizardStyle=modern
     - DisableProgramGroupPage=auto

  2. **[Files] section:**
     - Source: "..\dist\Astra\*"; DestDir: "{app}"; Flags: ignoreversion recursesubdirs createallsubdirs
     - This copies the entire --onedir output into the install directory

  3. **[Icons] section (INST-02, INST-03):**
     - Desktop shortcut: Name: "{userdesktop}\Astra Interview Copilot"; Filename: "{app}\Astra.exe"; Tasks: desktopicon
     - Start Menu: Name: "{group}\Astra Interview Copilot"; Filename: "{app}\Astra.exe"
     - Start Menu uninstall: Name: "{group}\Uninstall Astra"; Filename: "{uninstallexe}"

  4. **[Tasks] section:**
     - Name: "desktopicon"; Description: "Create a desktop shortcut"; GroupDescription: "Additional icons:"

  5. **[UninstallDelete] section (INST-08: preserve user data):**
     - Do NOT delete {localappdata}\astra\ (ChromaDB, config, prompts)
     - Only clean up the install directory {app}
     - Add comment explaining user data preservation

  6. **[Code] section (optional):**
     - Add a Pascal Script function to check if the app is running before install/uninstall
     - Use FindWindowByClassName or similar to detect running instance
     - Show message "Please close Astra before continuing" if running

  7. **Progress bar (INST-07):**
     - Inno Setup shows progress bar by default during file extraction — no extra config needed

  Key Inno Setup syntax notes:
  - Use {localappdata} NOT {pf} for per-user install
  - PrivilegesRequired=lowest is the key setting for no-admin
  - LZMA2 compression keeps installer small
  - The [UninstallRun] section is NOT needed — we just don't delete user data dirs

  Do NOT include any auto-update functionality (out of scope).
  Do NOT include splash screens or custom wizard pages beyond standard flow.
  </action>
  <verify>
  - installer/astra_setup.iss exists and contains valid Inno Setup syntax
  - PrivilegesRequired=lowest is set
  - Desktop shortcut and Start Menu entries defined
  - Source path references dist/Astra/ (matching --onedir output)
  - No references to {pf} or {commonpf} (admin-required paths)
  - User data directory (%APPDATA%/astra/) is NOT in any delete section
  </verify>
  <done>
  Inno Setup script creates a per-user installer with desktop shortcut, Start Menu entry, and clean uninstall that preserves user data.
  </done>
