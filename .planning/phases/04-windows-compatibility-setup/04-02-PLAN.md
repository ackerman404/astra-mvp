  <action>
Add `WindowsAudioCapture(AudioCapture)` class to audio_capture.py:

1. **Import PyAudioWPatch conditionally**:
   ```python
   if sys.platform == "win32":
       import pyaudiowpatch as pyaudio
   ```

2. **Implement `WindowsAudioCapture`**:
   - `__init__(device=None)`: Initialize PyAudio, find loopback device
   - `_find_loopback_device()`: Find WASAPI loopback device for speakers
   - `start_capture()`: Open input stream from loopback device
   - `stop_capture() -> np.ndarray`: Close stream, return buffer
   - `get_last_n_seconds(n) -> np.ndarray`: Return recent buffer slice
   - `get_audio_level() -> float`: Calculate RMS from recent samples
   - `list_devices() -> list[dict]`: Enumerate WASAPI devices

3. **Device detection for Windows**:
   - Get default output device
   - Find its loopback counterpart (same name + " [Loopback]")
   - PyAudioWPatch adds virtual loopback devices at end of list

4. **Buffer handling**:
   - Use same buffer size as Linux (60 seconds at 16kHz)
   - Convert audio format to match Whisper expectations (16kHz mono int16)
   - Use threading for background capture (same pattern as Linux)

5. **Update factory function**:
   ```python
   def get_audio_capture(device=None):
       if sys.platform == "win32":
           return WindowsAudioCapture(device)
       elif sys.platform.startswith("linux"):
           return LinuxAudioCapture(device)
       else:
           raise NotImplementedError(f"Unsupported platform: {sys.platform}")
   ```

Reference PyAudioWPatch docs for WASAPI loopback pattern:
- Use `p.get_wasapi_loopback_analogue_by_dict()` to find loopback device
- Or iterate devices looking for loopback flag
