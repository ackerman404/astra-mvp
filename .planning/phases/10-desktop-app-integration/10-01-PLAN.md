<task type="auto">
  <name>Task 1: Add /v1/embeddings proxy endpoint to backend</name>
  <files>backend/proxy.py, backend/config.py</files>
  <action>
  **backend/config.py:**
  Add `ALLOWED_EMBEDDING_MODELS: list[str] = ["text-embedding-3-small"]` to the Settings class. Keep the existing ALLOWED_MODELS for chat completions unchanged.

  **backend/proxy.py:**
  Add a new endpoint `POST /v1/embeddings` that mirrors the pattern of the existing `/v1/chat/completions` endpoint:

  1. Add `@router.post("/v1/embeddings")` with the same license validation dependency (`check_rate_limit`)
  2. Parse the request body as JSON (pass-through, same pattern as chat completions)
  3. Validate `model` against `settings.ALLOWED_EMBEDDING_MODELS` (not the chat ALLOWED_MODELS)
  4. Forward to OpenAI via `client.embeddings.create(**body)` using the shared `openai_client` from `request.app.state`
  5. Return the response as JSON via `response.model_dump()`
  6. Add the same error mapping pattern (AuthenticationError→502, RateLimitError→429, etc.)
  7. Add fire-and-forget usage logging with endpoint="embeddings"
  8. Add retry logic: single retry on 429/500 with 2s backoff (create `_call_embeddings_with_retry` helper, same pattern as `_call_openai_with_retry`)

  Embeddings are non-streaming, so no SSE handling needed. Much simpler than chat completions.

  **Important:** The endpoint path must be exactly `/v1/embeddings` because the OpenAI SDK constructs this path from `base_url + /embeddings`. When a desktop client uses `OpenAI(base_url="http://proxy:8000/v1")`, the SDK calls `http://proxy:8000/v1/embeddings`.
  </action>
  <verify>
  Start the backend server and test with curl:
  ```

  # Get the key from the output
  # Activate it

  # Test embeddings endpoint

  ```

<task type="auto">
  <name>Task 2: Replace API key config with license key + proxy URL config</name>
  <files>config.py</files>
  <action>
  Replace the `get_api_key()` function and add new config functions for the proxy architecture. The config file at `~/.config/astra/.env` will store LICENSE_KEY and PROXY_URL instead of OPENAI_API_KEY.

  **Replace `get_api_key()` with `get_license_key()`:**
  - Same implementation pattern but reads `LICENSE_KEY=` instead of `OPENAI_API_KEY=`
  - Keep the same .env file location (platformdirs)
  - Return None if not found

  **Add `save_license_key(key: str) -> bool`:**
  - Write `LICENSE_KEY=<key>` to the .env file
  - Preserve other lines in the file (like PROXY_URL)
  - Return True on success, False on failure

  **Add `clear_license_key() -> bool`:**
  - Remove the LICENSE_KEY line from the .env file
  - Return True on success

  **Add `get_proxy_url() -> str`:**
  - Read `PROXY_URL=` from the .env file
  - Default to `https://astra-proxy.up.railway.app/v1` if not set (production default)
  - For local dev, user can set `PROXY_URL=http://localhost:8000/v1`

  **Add `save_proxy_url(url: str) -> bool`:**
  - Write `PROXY_URL=<url>` to the .env file
  - Preserve other lines

  **Add `get_hardware_id() -> str`:**
  - Generate a stable machine identifier:
    - Linux: Read `/etc/machine-id` and hash it with SHA-256
    - Windows: Run `wmic csproduct get uuid`, parse output, hash with SHA-256
    - Fallback: `hashlib.sha256(str(uuid.getnode()).encode()).hexdigest()[:32]`
  - Return a hex string (32 chars)
  - Cache the result (compute once per process)

  **Remove `get_api_key()`** entirely — it should not exist anymore. Also remove `get_config_path()` if it only served the API key display. If `get_config_path()` is used elsewhere, keep it but update the docstring.

  **Keep `get_config_dir()` and all prompts config functions unchanged.**

  **Helper for .env read/write:**
  Create a small internal helper `_read_env_file() -> dict[str, str]` and `_write_env_file(data: dict[str, str]) -> bool` to avoid duplicating the parse/write logic across get/save functions. The .env format is simple KEY=value lines.
  </action>
